# Pilot Selection Guide

## Purpose

This guide determines which consequence-boundary corridor should be piloted first.

The goal is not to choose the most impressive corridor.

The goal is to choose the corridor where consequence is concrete, integration is narrow, standing can be evaluated, and proof can be inspected.

## Pilot standard

A corridor is ready for first pilot when it has:

```text
one protected action class
one execution boundary
one governing layer
one outcome model
one physical posture map
one receipt/replay path
one acceptance condition
```

## Selection criteria

| Criterion | Meaning | Why it matters |
|---|---|---|
| Consequence clarity | The protected effect is specific | Avoids abstract governance claims |
| Buyer pain | Failure is expensive or legally exposed | Drives enterprise urgency |
| Boundary location | The commit point can be identified | Makes enforcement testable |
| Governing inputs | Required authority/evidence/state sources exist | Prevents speculative policy claims |
| Outcome posture | EXECUTE/NARROW/ESCALATE/REFUSE maps to physical behavior | Proves the boundary controls effect |
| Receipt value | A buyer benefits from proof and replay | Creates audit/compliance value |
| Integration narrowness | Pilot can run without replacing the workflow | Reduces adoption friction |
| Protected IP discipline | Public proof does not expose production substrate | Preserves commercial value |

## Corridor ranking for first pilot

### 1. Privacy Action Custody

**Rank:** First pilot candidate.

**Why:** Legal standing, runtime state, and consequence already collide in privacy workflows.

Protected actions are easy to define:

```text
cross-border transfer
third-party disclosure
AI training-data use
high-risk data sharing
retention/deletion action
```

Best first protected action:

```text
High-risk personal data transfer before effect.
```

### 2. AI Output-to-Action

**Rank:** Strong second pilot.

**Why:** Enterprises are deploying AI into workflows faster than they can prove whether output should become action.

Best first protected action:

```text
Model-generated recommendation becoming an operational workflow update.
```

### 3. Enterprise Access / Privileged Operations

**Rank:** Strong security/compliance corridor.

**Why:** Authentication proves requester identity, not runtime standing.

Best first protected action:

```text
Privileged production action under live authority and risk state.
```

### 4. Aerospace / Engineering Stage Admission

**Rank:** High-prestige evaluation corridor, not first commercial default.

**Why:** Strong category expansion, but review cycles are longer and claims must remain tightly bounded.

Best first protected action:

```text
AI-assisted engineering note advancing to simulation admission.
```

### 5. AI Memory / Identity / Agent Continuity

**Rank:** Strategic future-facing corridor.

**Why:** Agent memory and identity persistence will become critical, but buyer pain is still emerging.

Best first protected action:

```text
Long-memory agent storing identity-bearing memory for future action context.
```

## Recommended first pilot

```text
Privacy Action Custody — High-Risk Personal Data Transfer Corridor
```

## Why this is the first commercial corridor

Privacy has:

```text
clear legal standing requirements
clear protected actions
clear buyer pain
clear audit value
clear consequence
clear need for receipt/replay
```

The strongest pilot claim:

```text
No high-risk data action reaches effect unless governed runtime custody resolves that the action still has standing.
```

## Pilot acceptance condition

```text
If a high-risk data action lacks standing, it must not reach effect.

The system must resolve EXECUTE, NARROW, ESCALATE, or REFUSE before the action binds, and the decision must be receipt-bound and replayable.
```

## Buyer discovery questions

1. Which high-risk data action creates the most exposure today?
2. Where does that action move from approval into actual execution?
3. Which systems hold lawful basis, consent, objection, transfer approval, DPIA, processor status, and incident hold state?
4. What should happen when standing is incomplete or stale?
5. Who owns the governing law layer?
6. Where should receipts be stored?
7. What replay evidence would satisfy legal, privacy, security, and audit stakeholders?

## Clean pilot line

```text
Start with one protected data action.
Prove that only actions with current standing reach effect.
Everything else narrows, escalates, refuses, or halts with receipt and replay.
```
